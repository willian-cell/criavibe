## Analise Tecnica

O projeto CriaVibe usa PHP nativo, HTML, CSS e JavaScript Vanilla. A configuracao anterior dependia obrigatoriamente do arquivo `.env`, o que quebraria em ambientes como Railway, onde as credenciais sao injetadas por variaveis de ambiente no servico.

O Railway MySQL fornece variaveis como `MYSQLHOST`, `MYSQLPORT`, `MYSQLUSER`, `MYSQLPASSWORD`, `MYSQLDATABASE` e `MYSQL_URL`. O sistema foi ajustado para aceitar essas variaveis sem exigir alteracao manual de codigo.

## Implementacao

- `api/lib/DotEnv.php`: o carregamento do `.env` passou a ser opcional.
- `api/config.php`: passou a resolver credenciais via `DB_*`, `MYSQL*`, `MYSQL_URL` ou `DATABASE_URL`.
- `Dockerfile`: criado para instalar `pdo_mysql` e executar o servidor PHP usando a porta definida pelo Railway.
- `router.php`: criado para rotear arquivos estaticos e APIs no servidor embutido do PHP.
- `env_example.txt`: removidas credenciais reais e documentadas variaveis esperadas em local/Railway.
- `saiba_mais.html`: `.parallax-bg` passou a conter um video em autoplay, mutado, loop e `playsinline`, mantendo imagem de fallback.
- `assets/videos/.gitkeep`: criado diretorio para armazenar o MP4 do hero.
- `api/db_migrations.php`: passou a criar o schema base completo quando o MySQL estiver vazio.
- `api/auth/register.php`: passou a retornar erro JSON claro caso o schema ainda nao tenha sido migrado.
- `index.html`: logo `logo/logo-criavibe-fotografia.jpeg` posicionada lado a lado com o titulo principal no hero.
